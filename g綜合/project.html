Why remote repositories?

  • Backups & history in the cloud.
  • Team collaboration with branches, PRs, reviews.
  • Deployment & automation (e.g., GitHub Pages).

Create a GitHub repository

  1. GitHub → New → name the repo (e.g., git-guide).
  2. Choose Public/Private, optionally initialize with README.
  3. Create repo and copy the HTTPS URL.

Create a local project & link to remote

mkdir git-guide
cd git-guide
git init
echo "# The Git Guide" > README.md
git add .
git commit -m "chore: init"
git branch -M main
git remote add origin https://github.com/<org>/<repo>.git
git push -u origin main

How others can contribute

# clone the repo
git clone https://github.com/<org>/<repo>.git
cd repo
# feature branch
git checkout -b feature/<name>-task
# work → stage → commit
git add .
git commit -m "feat: add X"
# push & open PR to develop
git push -u origin feature/<name>-task
