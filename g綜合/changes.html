Changes — Daily Workflow

1) Pull latest base (develop)

git checkout develop
git pull

2) Create a feature branch

git checkout -b feature/<your-name>-changes

3) Edit → Stage → Commit

git status
git add .
git commit -m "feat: implement changes page"

4) Push & open PR → develop

git push -u origin feature/<your-name>-changes
PR tip: Title with type (feat/fix/docs), describe what & why, and add how-to-test + screenshots.

Keep your branch fresh

git checkout develop && git pull
git checkout feature/<your-name>-changes
git merge develop
# resolve conflicts:
# edit files to keep the right content, remove conflict markers
git add <file>
git commit -m "chore: resolve merge conflict"
git push

Common errors

Push rejected: run git pull, resolve conflicts, then git push.

Cannot push to main: main may be protected; open a PR into develop.

Committed on wrong branch: switch to the correct branch and redo your change there.

Personal “9 actions” checklist

  • Create a feature branch
  • Make 2–3 small commits
  • Push the branch
  • Open a PR to develop
  • Request reviews & respond
  • Merge into develop
  • Review ≥3 teammates’ PRs
