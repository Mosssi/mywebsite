Branching

Why branching?

  • Isolate work; keep main stable.
  • Parallel development & safer reviews.
  • Typical flow: feature → develop → main.

Create & switch branches

git branch                  # list
git checkout -b feature/<name>-topic
git checkout develop        # switch back

Push & PR

git push -u origin feature/<name>-topic
# open a PR to develop

Merge a branch

git checkout develop
git pull
git merge feature/<name>-topic
git push

Delete a branch (after merge)

git branch -d feature/<name>-topic
git push origin --delete feature/<name>-topic
